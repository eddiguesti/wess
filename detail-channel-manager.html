03 — Deep Dive

Channel Manager
Centralisation

One place to manage all rates, pushed everywhere simultaneously. No more individual spreadsheets for each tour operator. No more rate discrepancies between channels. One rate, everywhere, always in sync.

The Vision

One Source of Truth for Every Rate

Rates currently live in multiple places: Opera, spreadsheets, emails, individual tour operator portals. When a rate changes, someone updates it everywhere manually. The channel manager becomes the single hub that connects every distribution channel and keeps everything in sync.

The end state: Your revenue manager sets a rate once in Opera. Within minutes, that rate is live on Booking.com, Expedia, your direct booking engine, and every connected tour operator portal. Availability updates in real time. No spreadsheets. No phone calls. No "which rate did we send them?"

Two-Way Synchronisation

This is not just a rate push. When a booking comes in from Booking.com, availability updates across all channels instantly. When Opera's inventory changes for any reason, every channel reflects it. Overbooking risk drops to near zero.

OTA Distribution

Online Channels, Automated

Booking.com

Rates, availability, and content managed through the channel manager. When you update a room description or add photos in Opera, the channel manager pushes those changes to Booking.com automatically.

Expedia Group

Expedia, Hotels.com, Vrbo. The entire Expedia Group connected through a single integration. Rate parity maintained across all sub-brands without manual intervention.

Additional OTAs can be connected as needed: Agoda, TripAdvisor, Google Hotels, and regional platforms specific to Mauritius tourism
Each channel can have its own rate rules (markup, minimum stay, close-out dates) managed from one interface
Tour Operator Portal Connections

API-Connected Tour Operators

For TOs That Accept API Connections

Larger tour operators (TUI, DER Touristik, FTI, and others) often have booking portals that accept API connections. For these operators, the channel manager pushes rates and availability directly into their systems. No email needed. No manual upload. The rate you set in Opera is the rate they see in their booking system within minutes.

The hybrid approach: Not every tour operator can connect via API. The channel manager handles those who can; the Email Agent (from Phase 1) handles those who can't. Between the two systems, every distribution channel is covered, whether it is a global OTA, an API-connected tour operator, or a small specialist operator who still works by email.

Complete Coverage

The Mauritius market includes dozens of smaller, specialist tour operators who work only via email. No booking portals. No API connections. The Email Agent handles these automatically. When rates change, it sends personalised rate sheets to every email-only operator. Same rate, different delivery method.

Rate Parity Monitoring

The Same Rate, Everywhere

Parity Dashboard

A live view showing the rate for every room type across every channel. If Booking.com is showing EUR 180 but Expedia is showing EUR 175, you see it immediately. Rate parity violations are flagged in real time, not discovered by a guest who found a cheaper price elsewhere.

Automatic Corrections

When a parity violation is detected, the system can either alert your team or automatically push the correct rate. Configurable per channel. Some channels you want to correct automatically, others you want to review first.

Rate parity is increasingly monitored by OTAs themselves. Booking.com and Expedia both penalise properties with parity violations through lower search rankings
Direct booking rates can be configured separately, offering a best-price guarantee on the WESS website to encourage direct bookings at a lower commission cost
What We Need from the Process Map

To Configure Distribution

Process Map Required

Channel manager configuration depends on understanding how WESS distributes rates and manages availability today. This comes from the process mapping phase.

API vs. email Which tour operators can connect via API? Which ones only work by email? This determines the split between channel manager and email agent.
Rate structures How are rates structured? Seasonal tiers (low/mid/high), contract rates per TO, promotional rates, rack rates? Each structure needs to be modelled in the channel manager.
Allotment management How do allotments work? Which TOs have guaranteed allotments? What are the release periods? How are free-sale and on-request rooms handled?
OTA contracts What are the current OTA commission structures? What content is managed where? What are the existing rate rules and restrictions?
Approval workflows Who approves rate changes? Can the revenue manager push rates directly, or does Peter need to sign off on certain changes?
